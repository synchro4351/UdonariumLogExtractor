# Udonarium Log Extractor

Udonariumの部屋データ（`.zip`）から、チャットログをテキストに抽出するローカル実行ツールです。

既存のWeb提供ツールだと処理データ量の上限が気になるため、ローカル環境で大量ログを扱えるように作っています。

## できること（初期の最小構成）

- `未処理` フォルダ内の `.zip` を順番に処理
- `chat.xml` から次の情報だけを抽出
  - タブ名
  - 発言者名
  - 発言内容
- 抽出結果を `出力ログ` に `.txt` で出力
- 正常終了した `.zip` を `処理済み` へ移動

## フォルダ構成

```text
.
├─ extract_udonarium_logs.py
├─ 未処理/      # 処理前の部屋データ(zip)を置く
├─ 処理済み/    # 処理後のzipが移動される
└─ 出力ログ/    # 抽出されたテキストログ
```

## 使い方

1. Python 3.10+ を用意
2. `未処理` フォルダにUdonariumの部屋データ（zip）を置く
3. 次を実行

```bash
python extract_udonarium_logs.py
```

処理が終わると、`出力ログ` にログテキストが作成され、対象zipは `処理済み` に移動します。

## オプション

```bash
python extract_udonarium_logs.py \
  --unprocessed-dir 未処理 \
  --processed-dir 処理済み \
  --output-dir 出力ログ
```

## 出力例

```text
元ファイル: sample_room.zip

=== タブ: メインタブ ===
GM: ではセッションを開始します
PL1: よろしくお願いします！

=== タブ: サブタブ ===
PL2: 相談用です
```

## 仕様メモ（今後変更しやすくするため）

- 現在は「タブごとに見出しを出して、その下に `発言者: 発言`」という単純形式
- ID・タイムスタンプなどは、意図的に出力していません

### タブの扱いの提案

今後仕様変更する前提として、次の2案が扱いやすいです。

1. タブ見出しセクション方式（現在）
   - 人間が読みやすい
   - 後で手作業編集しやすい
2. 1行プレフィックス方式
   - 例: `[メインタブ] GM: 発言`
   - grepや集計スクリプトで機械処理しやすい

必要なら次の段階で、CLIオプション（`--tab-format section|prefix`）として両対応にできます。
